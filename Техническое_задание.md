# Техническое задание
## Проект: анализ веб-приложения Demoblaze (интернет-магазин электроники)

### 1. Общая информация
Название приложения: Demoblaze  
Ссылка: https://www.demoblaze.com  
Тип проекта: веб-приложение (интернет-магазин)

Demoblaze — демонстрационный интернет-магазин электроники, который позволяет пользователю просматривать товары по категориям, изучать карточки товаров, добавлять товары в корзину и оформлять заказ.

---

### 2. Цель проекта
Цель проекта — выполнить анализ функциональности веб-приложения Demoblaze и подготовить документацию (README и техническое задание) для дальнейших работ по тестированию и описанию требований.

---

### 3. Задачи сайта (что должно обеспечивать приложение)
Сайт/приложение обеспечивает пользователю следующие возможности:
- просмотр каталога товаров;
- переключение категорий товаров (Phones, Laptops, Monitors);
- просмотр карточки товара (название, цена, описание, изображение);
- регистрация пользователя;
- авторизация пользователя;
- добавление товаров в корзину;
- просмотр корзины и управление составом (удаление товара);
- оформление заказа через форму (ввод данных покупателя);
- отправка обращения через раздел Contact (если используется).

---

### 4. Целевая аудитория
Целевая аудитория веб-приложения:
1. Пользователи, которые хотят выбрать и купить технику онлайн (телефоны, ноутбуки, мониторы).
2. Пользователи, сравнивающие товары по цене и описанию перед покупкой.
3. Новые пользователи, которые хотят быстро зарегистрироваться и оформить заказ без сложных шагов.

---

### 5. Проблемы, которые решает приложение
Приложение решает следующие задачи/проблемы пользователя:
- упрощает поиск и выбор товара по категориям;
- позволяет быстро получить информацию о товаре (цена, описание);
- даёт возможность собрать набор товаров в корзину;
- позволяет оформить заказ онлайн без визита в магазин;
- сокращает время покупки по сравнению с офлайн-выбором.

Аналоги:

- Demoblaze предоставляет базовый набор функций интернет-магазина (каталог → товар → корзина → оформление), поэтому удобен как учебный пример для анализа и тестирования.

---

### 7. Описание работы приложения (основные сценарии)
Сценарий 1: Просмотр каталога
1) Пользователь открывает главную страницу.  
2) Выбирает категорию товара.  
3) Просматривает список товаров и переходит в карточку.

Сценарий 2: Регистрация и вход
1) Пользователь открывает форму Sign up и создаёт аккаунт.  
2) Выполняет вход через Log in.

Сценарий 3: Добавление в корзину
1) В карточке товара нажимает Add to cart.  
2) Переходит в Cart и видит список добавленных товаров.  
3) При необходимости удаляет товар.

Сценарий 4: Оформление заказа
1) В корзине нажимает Place Order.  
2) Заполняет данные в форме.  
3) Подтверждает заказ.
